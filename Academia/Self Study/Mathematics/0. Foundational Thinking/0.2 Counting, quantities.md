### **0.2.1 Counting**

Counting is the process of assigning numbers to objects **one-by-one** to measure quantity.

Example:  
If you have apples and you count  
1 → first apple  
2 → second apple  
3 → third apple

The **last number spoken** tells you the total quantity.

### **0.2.2 Cardinality**

Cardinality means:

> _The final counting number equals the number of objects._

If you count to **5**, there are exactly **5 objects**, regardless of size, shape, or order.

### **0.2.3 Order does NOT change quantity**

These two counts give the same quantity:

- Count left to right → 6 objects
- Count right to left → 6 objects

The order of counting does not affect the total.

### **0.2.4 One-to-one correspondence**

Each object must be matched with **exactly one number**.

Correct counting:

- One object → one number

Incorrect counting:

- Skipping numbers
- Counting the same object twice

### **0.2.5 Counting different types of objects**

Quantity ignores type.

Example:

- 2 books + 3 pens = **5 objects**

The number only measures “how many,” not “what kind.”
