## Motivation

Most ODEs cannot be solved analytically. Numerical methods approximate solutions at discrete points.

### Initial Value Problem

$$y' = f(x, y), \quad y(x_0) = y_0$$

Goal: Approximate $y(x)$ at points $x_0, x_1, \ldots, x_N$ where $x_i = x_0 + ih$.

---

## Euler's Method

### Formula

$$y_{n+1} = y_n + hf(x_n, y_n)$$

where $h$ is step size.

### Geometric Interpretation

Follow tangent line with slope $f(x_n, y_n)$ for distance $h$.

```desmos-graph
y = e^x
y = 1 + x
y = 1 + x + x^2/2
```

> [!example] Visual Interpretation
> Euler's method follows piecewise linear approximation to true solution curve.

### Example

Solve $y' = x + y$ with $y(0) = 1$, step size $h = 0.1$:

- $x_0 = 0$, $y_0 = 1$
- $y_1 = 1 + 0.1(0 + 1) = 1.1$
- $y_2 = 1.1 + 0.1(0.1 + 1.1) = 1.22$
- Continue...

### Error Analysis

**Local truncation error**: $O(h^2)$ per step

**Global error**: $O(h)$ after $N$ steps

First-order method (low accuracy).

---

## Improved Euler (Heun's Method)

### Predictor-Corrector

**Predictor** (Euler): $\tilde{y}_{n+1} = y_n + hf(x_n, y_n)$

**Corrector**: $y_{n+1} = y_n + \frac{h}{2}[f(x_n, y_n) + f(x_{n+1}, \tilde{y}_{n+1})]$

Average slopes at beginning and predicted end.

### Accuracy

**Global error**: $O(h^2)$

Second-order method (better than Euler).

---

## Runge-Kutta Methods

### RK4 (Classical Fourth-Order)

Most popular numerical method:

$$\begin{aligned} k_1 &= f(x_n, y_n) \\ k_2 &= f(x_n + h/2, y_n + hk_1/2) \\ k_3 &= f(x_n + h/2, y_n + hk_2/2) \\ k_4 &= f(x_n + h, y_n + hk_3) \\ y_{n+1} &= y_n + \frac{h}{6}(k_1 + 2k_2 + 2k_3 + k_4) \end{aligned}$$

### Accuracy

**Global error**: $O(h^4)$

Fourth-order method (high accuracy).

### Why RK4?

- Good balance of accuracy and computational cost
- No need to compute derivatives of $f$
- Stable for many problems

## Adaptive Step Size

### Motivation

Fixed step size may be inefficient:
- Too small: unnecessary computation
- Too large: poor accuracy or instability

### Embedded Methods

Use two methods of different orders to estimate error.

Example: RK45 uses 4th and 5th order Runge-Kutta.

### Step Size Control

If estimated error $> $ tolerance: reduce $h$

If estimated error $\ll$ tolerance: increase $h$

## Systems of ODEs

### Vector Form

For $\mathbf{y}' = \mathbf{f}(x, \mathbf{y})$:

All methods apply component-wise.

### Example: Second-Order Equation

Convert $y'' = f(x, y, y')$ to system:

Let $u = y$, $v = y'$:
$$\begin{aligned} u' &= v \\ v' &= f(x, u, v) \end{aligned}$$

Apply numerical method to vector $(u, v)$.

## Stability Analysis

### Test Equation

$$y' = \lambda y, \quad y(0) = 1$$

Exact solution: $y = e^{\lambda x}$

### Absolute Stability

Method is **absolutely stable** if numerical solution doesn't grow when exact solution decays.

For $\text{Re}(\lambda) < 0$, need $|y_{n+1}| \leq |y_n|$.

### Stability Regions

|Method|Stability Condition|
|---|---|
|Euler|$|1 + h\lambda| \leq 1$|
|RK4|$|1 + h\lambda + \frac{(h\lambda)^2}{2} + \frac{(h\lambda)^3}{6} + \frac{(h\lambda)^4}{24}| \leq 1$|

## Stiff Equations

### Definition

**Stiff equation**: Has components that decay at very different rates.

Example: $y' = -1000y + \sin(x)$

Fast decay term requires tiny step size for explicit methods.

### Implicit Methods

**Backward Euler**: $y_{n+1} = y_n + hf(x_{n+1}, y_{n+1})$

Implicit (must solve equation for $y_{n+1}$).

**Advantage**: A-stable (stable for all $h$ when $\text{Re}(\lambda) < 0$).

### Trapezoidal Method

$$y_{n+1} = y_n + \frac{h}{2}[f(x_n, y_n) + f(x_{n+1}, y_{n+1})]$$

Also implicit, also A-stable.

---

## Boundary Value Problems

### Shooting Method

Convert BVP to sequence of IVPs:

1. Guess initial slope $y'(x_0) = s$
2. Solve IVP numerically
3. Check if boundary condition at other end is satisfied
4. Adjust $s$ using root-finding (e.g., bisection, Newton's method)

### Finite Difference Method

Discretize domain, replace derivatives with finite differences.

Example: $y'' = f(x,y)$ becomes:
$$\frac{y_{i+1} - 2y_i + y_{i-1}}{h^2} = f(x_i, y_i)$$

Gives system of algebraic equations.

---

## Error Types

### Truncation Error

Error from approximating derivative/integral.

Depends on method order and step size.

### Roundoff Error

Error from finite precision arithmetic.

Becomes significant for very small $h$ or many steps.

### Total Error

Combination of truncation and roundoff errors.

Optimal $h$ balances both.

---

## Common Mistakes

> [!warning] Using Too Large Step Size
> Large $h$ gives inaccurate results or instability. Check convergence by halving $h$.

> [!warning] Ignoring Stiffness
> Explicit methods fail for stiff equations. Use implicit methods instead.

> [!warning] Not Checking Conservation Laws
> For physical systems, verify energy/momentum conservation as sanity check.

1. Apply Euler's method with $h=0.2$ to approximate $y(1)$ for $y' = y$, $y(0) = 1$. Compare with exact value $e$.
2. Use improved Euler method with $h=0.1$ to solve $y' = x - y$, $y(0) = 1$ for three steps.
3. Implement one step of RK4 for $y' = y\cos(x)$, $y(0) = 1$ with $h=0.1$.
4. Convert the second-order equation $y'' + y' - 2y = 0$ to a first-order system.
5. Determine the stability region for backward Euler method applied to $y' = \lambda y$.
6. Explain why explicit methods struggle with stiff equations.
7. Describe how the shooting method works for BVPs.
8. Compare the computational cost of Euler's method vs RK4 for the same accuracy.
